# Topic 5: Computer Networks & Security (60 Q)

<details>
<summary><b>Q: OSI model layers (7)?</b></summary>
<blockquote><b>Answer:</b> Physical, Data Link, Network, Transport, Session, Presentation, Application</blockquote>
</details>

<details>
<summary><b>Q: TCP/IP model layers (4)?</b></summary>
<blockquote><b>Answer:</b> Link, Internet, Transport, Application</blockquote>
</details>

<details>
<summary><b>Q: IP address (IPv4)?</b></summary>
<blockquote><b>Answer:</b> 32-bit, 4 octets. Format: xxx.xxx.xxx.xxx (0-255 each)</blockquote>
</details>

<details>
<summary><b>Q: Subnet mask?</b></summary>
<blockquote><b>Answer:</b> Identifies network and host portions. Example: 255.255.255.0 (/24)</blockquote>
</details>

<details>
<summary><b>Q: Router function?</b></summary>
<blockquote><b>Answer:</b> Forwards packets between networks based on IP addresses</blockquote>
</details>

<details>
<summary><b>Q: Switch function?</b></summary>
<blockquote><b>Answer:</b> Connects devices within network using MAC addresses (Layer 2)</blockquote>
</details>

<details>
<summary><b>Q: TCP vs UDP?</b></summary>
<blockquote><b>Answer:</b> TCP: connection-oriented, reliable, ordered. UDP: connectionless, unreliable, fast</blockquote>
</details>

<details>
<summary><b>Q: DNS function?</b></summary>
<blockquote><b>Answer:</b> Translates domain names to IP addresses. Example: google.com → 142.250.185.46</blockquote>
</details>

<details>
<summary><b>Q: TCP 3-way handshake?</b></summary>
<blockquote><b>Answer:</b> SYN → SYN-ACK → ACK (establishes connection)</blockquote>
</details>

<details>
<summary><b>Q: ARP (Address Resolution Protocol)?</b></summary>
<blockquote><b>Answer:</b> Maps IP addresses to MAC addresses on local network</blockquote>
</details>

<details>
<summary><b>Q: ICMP purpose?</b></summary>
<blockquote><b>Answer:</b> Internet Control Message Protocol. Diagnostics. Used by ping command</blockquote>
</details>

<details>
<summary><b>Q: Routing?</b></summary>
<blockquote><b>Answer:</b> Process of forwarding packets to destination. Uses routing table</blockquote>
</details>

<details>
<summary><b>Q: BGP (Border Gateway Protocol)?</b></summary>
<blockquote><b>Answer:</b> Routing protocol between autonomous systems on internet</blockquote>
</details>

<details>
<summary><b>Q: OSPF (Open Shortest Path First)?</b></summary>
<blockquote><b>Answer:</b> Interior routing protocol using link state algorithm</blockquote>
</details>

<details>
<summary><b>Q: DHCP function?</b></summary>
<blockquote><b>Answer:</b> Automatically assigns IP addresses to devices</blockquote>
</details>

<details>
<summary><b>Q: Firewall?</b></summary>
<blockquote><b>Answer:</b> Filters incoming/outgoing traffic based on rules</blockquote>
</details>

<details>
<summary><b>Q: Encryption purpose?</b></summary>
<blockquote><b>Answer:</b> Converts plaintext to ciphertext (unreadable without key)</blockquote>
</details>

<details>
<summary><b>Q: VPN (Virtual Private Network)?</b></summary>
<blockquote><b>Answer:</b> Encrypted tunnel over public network. Hides IP, encrypts data</blockquote>
</details>

<details>
<summary><b>Q: Symmetric encryption?</b></summary>
<blockquote><b>Answer:</b> Same key for encrypt/decrypt. Fast, secure but key distribution problem</blockquote>
</details>

<details>
<summary><b>Q: Asymmetric encryption?</b></summary>
<blockquote><b>Answer:</b> Public and private key pair. Slow, but solves key distribution</blockquote>
</details>

<details>
<summary><b>Q: RSA algorithm?</b></summary>
<blockquote><b>Answer:</b> Asymmetric encryption using two large prime numbers. Security: difficulty of factoring</blockquote>
</details>

<details>
<summary><b>Q: Digital signature?</b></summary>
<blockquote><b>Answer:</b> Ensures authenticity and non-repudiation. Created with private key, verified with public</blockquote>
</details>

<details>
<summary><b>Q: IPSec modes?</b></summary>
<blockquote><b>Answer:</b> Transport mode (end-to-end), Tunnel mode (network-to-network)</blockquote>
</details>

<details>
<summary><b>Q: SSL/TLS?</b></summary>
<blockquote><b>Answer:</b> Security protocols for web (HTTPS). Establishes encrypted connection</blockquote>
</details>

<details>
<summary><b>Q: HTTP vs HTTPS?</b></summary>
<blockquote><b>Answer:</b> HTTP: plain text. HTTPS: encrypted with SSL/TLS</blockquote>
</details>

<details>
<summary><b>Q: Congestion control?</b></summary>
<blockquote><b>Answer:</b> Prevents network overload. TCP uses window size adjustment, packet loss detection</blockquote>
</details>

<details>
<summary><b>Q: Quality of Service (QoS)?</b></summary>
<blockquote><b>Answer:</b> Ensures priority and bandwidth for critical traffic</blockquote>
</details>

<details>
<summary><b>Q: Bandwidth?</b></summary>
<blockquote><b>Answer:</b> Maximum data transfer rate (bits/second). Mbps, Gbps</blockquote>
</details>

<details>
<summary><b>Q: Latency?</b></summary>
<blockquote><b>Answer:</b> Time for packet to travel (milliseconds). Lower is better</blockquote>
</details>

<details>
<summary><b>Q: MAC address?</b></summary>
<blockquote><b>Answer:</b> 48-bit physical address. Format: 6 hex pairs. Layer 2 identifier</blockquote>
</details>

<details>
<summary><b>Q: Ethernet?</b></summary>
<blockquote><b>Answer:</b> IEEE 802.3. LAN technology using CSMA/CD</blockquote>
</details>

<details>
<summary><b>Q: CSMA/CD?</b></summary>
<blockquote><b>Answer:</b> Carrier Sense Multiple Access with Collision Detection</blockquote>
</details>

<details>
<summary><b>Q: WiFi (IEEE 802.11)?</b></summary>
<blockquote><b>Answer:</b> Wireless LAN technology. Uses 2.4GHz or 5GHz bands</blockquote>
</details>

<details>
<summary><b>Q: Port number?</b></summary>
<blockquote><b>Answer:</b> 16-bit identifier for service/process. Range: 0-65535</blockquote>
</details>

<details>
<summary><b>Q: Common ports?</b></summary>
<blockquote><b>Answer:</b> HTTP(80), HTTPS(443), FTP(21), SMTP(25), POP3(110), DNS(53)</blockquote>
</details>

<details>
<summary><b>Q: FTP (File Transfer Protocol)?</b></summary>
<blockquote><b>Answer:</b> Transfer files. Uses 2 connections: control (21), data (20)</blockquote>
</details>

<details>
<summary><b>Q: SMTP (Simple Mail Transfer Protocol)?</b></summary>
<blockquote><b>Answer:</b> Sends emails. Port 25. SMTP client = sender, SMTP server = receiver</blockquote>
</details>

<details>
<summary><b>Q: POP3 vs IMAP?</b></summary>
<blockquote><b>Answer:</b> POP3: downloads emails. IMAP: accesses remotely</blockquote>
</details>

<details>
<summary><b>Q: HTTP status codes?</b></summary>
<blockquote><b>Answer:</b> 200 (OK), 301 (redirect), 400 (bad request), 404 (not found), 500 (server error)</blockquote>
</details>

<details>
<summary><b>Q: Network topology: Star?</b></summary>
<blockquote><b>Answer:</b> All devices connected to central hub/switch</blockquote>
</details>

<details>
<summary><b>Q: Network topology: Bus?</b></summary>
<blockquote><b>Answer:</b> All devices share single cable (backbone)</blockquote>
</details>

<details>
<summary><b>Q: Network topology: Ring?</b></summary>
<blockquote><b>Answer:</b> Devices in circle. Token Ring passes token</blockquote>
</details>

<details>
<summary><b>Q: Network topology: Mesh?</b></summary>
<blockquote><b>Answer:</b> Multiple paths between devices. Redundant, reliable</blockquote>
</details>

<details>
<summary><b>Q: WAN (Wide Area Network)?</b></summary>
<blockquote><b>Answer:</b> Covers large geographic area. Lower speed, higher cost</blockquote>
</details>

<details>
<summary><b>Q: LAN (Local Area Network)?</b></summary>
<blockquote><b>Answer:</b> Covers small area (building). High speed, low cost</blockquote>
</details>

<details>
<summary><b>Q: MAN (Metropolitan Area Network)?</b></summary>
<blockquote><b>Answer:</b> Covers city area. Medium speed and cost</blockquote>
</details>
